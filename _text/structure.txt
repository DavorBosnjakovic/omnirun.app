# Omnirun App - Current Folder Structure
# Last Updated: April 6, 2026

## Project Location
D:\omnirun.app\app\

## Tech Stack
- Framework: Tauri (Rust backend, web frontend)
- Frontend: React + TypeScript
- Bundler: Vite
- Styling: Tailwind CSS v3
- State: Zustand
- Icons: Lucide React
- Terminal: xterm.js + @xterm/addon-fit
- Local Database: SQLite via tauri-plugin-sql
- Cloud Backend: Supabase (auth, profiles, subscriptions, analytics)
- Auth: @supabase/supabase-js (email/password, magic link, JWT session persistence in SQLite)

## SQLite Database (omnirun.db) -- Schema v8

| Table | Purpose |
|-------|---------|
| schema_version | Tracks migration version |
| projects | Saved projects (id, name, path, last_opened) |
| chat_history | Per-project chat conversations (messages JSON) |
| settings | Key-value store for all app settings |
| usage | Per-API-call token + cost tracking |
| usage_budget | Monthly budget settings (singleton) |
| usage_sessions | Session-level usage summaries |
| connections | Global project connections (provider, token, status) |
| project_connections | Per-project connections (project_id + provider) |
| auth_tokens | Supabase JWT session persistence (singleton) |
| assistant_accounts_cache | Local cache of Supabase assistant_email_accounts |
| notifications_cache | Local cache of unread notifications from watching agents (v7) |
| user_memory | User context file content -- singleton row (v8) |
| memory_observations | Raw observation log from conversation extraction (v8) |
| memory_settings | Memory toggle preferences -- singleton row (v8) |

### Supabase Edge Functions

| Function | Trigger | Purpose |
|----------|---------|---------|
| send-invitation-email | Database webhook on team_invitations INSERT + direct invoke from teamStore | Sends branded invitation email via Resend API (team name, inviter name, accept link, download link, 7-day expiry). Deployed --no-verify-jwt. Secrets: RESEND_API_KEY |
| stripe-create-checkout | App invokes from BillingSettings | Creates Stripe Checkout session for subscription signup. Raw fetch to Stripe API (no SDK). Deployed --no-verify-jwt. Secrets: STRIPE_SECRET_KEY |
| stripe-webhook | Stripe webhook events | Handles checkout.session.completed, subscription created/updated/deleted, invoice succeeded/failed. Raw fetch + Web Crypto HMAC-SHA256 sig verification. Deployed --no-verify-jwt. Secrets: STRIPE_SECRET_KEY, STRIPE_WEBHOOK_SECRET |
| stripe-create-portal | App invokes from BillingSettings | Creates Stripe Customer Portal session for managing subscriptions. Raw fetch to Stripe API. Deployed --no-verify-jwt. Secrets: STRIPE_SECRET_KEY |

### Supabase Database Triggers

| Trigger | Table | Event | Function | Purpose |
|---------|-------|-------|----------|---------|
| on_project_unlock | project_locks | AFTER DELETE | notify_project_available() | Inserts "X is now available" notification for all team members except the lock holder |
| on_invitation_created | team_invitations | AFTER INSERT | notify_invitation_created() | Inserts "You've been invited to join X" notification if invitee already has an account |
